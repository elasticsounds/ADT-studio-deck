What We Heard

Priority user feedback.

As communicated by country pilots and the user community — the signals shaping what we build.

Accessibility throughout
Pilots showed the tool itself must be accessible. A blind user should open ADT Studio, upload a PDF and get a WCAG-compliant output — useful and reputationally vital.
Flexible activity creation
Interactive activities that are easy to generate, convert and edit. Every education-publishing stakeholder wants a simpler path to editable activities.
Brand consistency
Control fonts and branded elements so outputs match the existing style of already-published material.
Matched to learning outcomes
Pedagogy and curriculum teams want to see how generated assets map to target learning outcomes — uploading the curriculum to give the system more context.
Tracking learning progress
Beyond SCORM/LMS sync or downloadable results, teachers need a simple way to understand and track learner progress.
Modular conversion
Break large textbooks into chapters — work one chapter at a time, or export smaller components.
